SQL with Command Prompt

USE DESCRIBE SHOW DATABASES SHOW TABLES CREATE TABLE Single Row INSERT Load Data from File Data Selection

What you'll learn: Get hands-on with MySQL using the command prompt — no GUI, just you and the terminal. You'll use USE db_name to select a database, SHOW DATABASES and SHOW TABLES to explore what exists, and DESCRIBE table_name to see a table's column structure. You'll also create your first real table with proper data types (INT, VARCHAR, DATE), insert a single row of data, and even load a bulk CSV file into MySQL using LOAD DATA INFILE. 🎬 Recorded session included.

4

Data Definition Language (DDL)

CREATE TABLE ADD MODIFY DROP DROP TABLE TRUNCATE DELETE

What you'll learn: DDL commands define and change the structure of your database — not the data inside it. You'll use CREATE TABLE to design a table from scratch, ADD to add a new column, and MODIFY to change a column's data type. You'll also understand a critical difference that trips up beginners: TRUNCATE vs DELETE — TRUNCATE wipes all rows instantly and can't be rolled back, while DELETE removes specific rows and can be undone. DROP TABLE removes the entire table permanently. 🎬 Recorded session included.

5

Data Control Language (DCL)

GRANT REVOKE

What you'll learn: In real companies, not everyone should have the same level of database access. DCL lets you control who can do what. GRANT gives a user specific permissions — for example, allowing a junior analyst to only SELECT (read) data without being able to modify or delete it. REVOKE takes those permissions away. You'll also learn about privilege levels — database-level, table-level, and column-level — which are essential knowledge for database admin and security roles. 🎬 Recorded session included.

6

Data Manipulation Language (DML)

INSERT UPDATE DELETE

What you'll learn: DML commands work on the data inside tables — not the structure. INSERT INTO adds new rows, UPDATE modifies existing rows (always use WHERE or you'll update every row!), and DELETE FROM removes specific rows. A key tip: always write and test your WHERE clause with a SELECT first before running UPDATE or DELETE — this is a professional habit that saves you from accidents in real databases. 🎬 Recorded session included.

UPDATE LIMIT

9

SQL with Python — Part 3

IN BETWEEN LIKE

What you'll learn: Three powerful filtering keywords used in almost every real-world SQL query. IN lets you match multiple values at once — e.g. WHERE city IN ('Mumbai', 'Delhi', 'Pune') instead of writing three OR conditions. BETWEEN filters a range — e.g. WHERE salary BETWEEN 30000 AND 60000. LIKE does pattern matching using wildcards: % matches any number of characters and _ matches exactly one — e.g. WHERE name LIKE 'A%' returns all names starting with A. All practised through Python. 🎬 Recorded session included.

10

Comparison Operators

=  Equal to >  Greater than <  Less than >=  Greater than or equal <=  Less than or equal <>  Not equal to

What you'll learn: Comparison operators are the backbone of every WHERE clause. = checks exact equality, <> (or !=) checks inequality. > and < filter ranges of numbers or dates. You'll practise with real examples like: find all employees with salary > 50000, find products where stock <= 10, find orders placed >= '2024-01-01'. These six operators appear in every single SQL query you'll ever write professionally. 🎬 Recorded session included.

11

Logical Operators

AND OR NOT

What you'll learn: Combine multiple conditions in one query. AND requires all conditions to be true — e.g. WHERE city = 'Delhi' AND salary > 40000. OR requires at least one condition to be true. NOT inverts a condition — e.g. WHERE NOT department = 'HR'. A key concept is operator precedence: SQL evaluates AND before OR, so always use parentheses to make your logic clear — e.g. WHERE (city = 'Delhi' OR city = 'Mumbai') AND salary > 50000. 🎬 Recorded session included.

12

Aggregate Functions

COUNT MIN MAX AVG SUM

What you'll learn: Aggregate functions let you summarise large amounts of data into a single meaningful number. COUNT(*) tells you how many rows exist, SUM(salary) totals all salaries, AVG(marks) calculates the average, MIN and MAX find the lowest and highest values. These are almost always used with GROUP BY — e.g. "total sales per region" or "average salary per department". You'll also learn COUNT(DISTINCT col) to count unique values — a very common interview question. 🎬 Recorded session included.

13

Joins

INNER JOIN LEFT JOIN RIGHT JOIN

What you'll learn: Joins are the most tested SQL topic in every data analyst interview — and for good reason. Real databases have data spread across multiple tables that you need to combine. INNER JOIN returns only rows where there's a match in both tables. LEFT JOIN returns all rows from the left table, filling NULL for rows that have no match on the right — useful when you want to see all customers, even those with no orders. RIGHT JOIN does the opposite. You'll practise on real table pairs like Customers + Orders and Employees + Departments. 🎬 Recorded session included.

14

Parent & Child Tables (Constraints)

Primary Key Foreign Key Parent Table Child Table UNIQUE Constraint

What you'll learn: Good database design prevents data errors before they happen. A Primary Key uniquely identifies every row in a table — no two rows can have the same value (e.g. Employee ID, Order ID). A Foreign Key links a child table back to a parent table — e.g. an Orders table has a customer_id Foreign Key that references the Customers table. This relationship ensures you can't add an order for a customer that doesn't exist. The UNIQUE constraint ensures a column (like Email) has no duplicates. These concepts directly enable Joins. 🎬 Recorded session included.

15

Sub Queries / Nested Queries / Inner Queries

Subquery Basics Nested SELECT Correlated Subqueries Subquery in WHERE Subquery in FROM

What you'll learn: A subquery is a query inside another query — it lets you answer complex questions in one step. Example: "Find all employees who earn more than the average salary" — you can't use AVG() directly in WHERE, but you can nest it: WHERE salary > (SELECT AVG(salary) FROM employees). You'll learn subqueries in WHERE (to filter using a computed value), in FROM (to create a virtual table), and correlated subqueries that reference the outer query row by row. This is an advanced topic that separates strong SQL writers from average ones. 🎬 Recorded session included.
